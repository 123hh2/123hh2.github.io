# JC0103. Symmetric Matrix

# 题目大意

给定 nnn 种 2*2 大小的矩阵，问是否能用以上任意种矩阵填满大小为 m∗mm * mm∗m 的矩阵使得该矩阵 sss 满足 s[i][j]=s[j][i]s[i][j]=s[j][i]s[i][j]=s[j][i]

# 思路

模拟填矩阵，发现填完之后的大矩阵是关于从左上到右下的斜对角线对称的，将这个对称性转移到每个小矩阵上可以看出小矩阵必须满足和大矩阵一样的对称性质，所以我们只需要找到一个这样的小矩阵就能构造出一个满足条件的大矩阵

显然，当 m 为奇数时无解

# 代码

# JC0104. Happy Birthday, Polycarp!

# 题目大意

求 [1,n][1,n][1,n] 中每位数字都相同的数的个数

# 思路

一眼模拟，发现对于不管几位数，都有 999 种每位数字都相同的数，所以答案为 nnn 的位数所对应的每位数字都相同的个数加上剩下位数的个数 * 9 即可

# 代码

# JC0105. A New Technique

# 题目大意

有一个 n∗mn * mn∗m 的矩阵，给出从左到右每一行元素和从上到下每一列的元素，但是行和列的顺序不一样，求原矩阵，保证每个元素都不一样

# 思路

既然每个元素都不一样，我们知道在某一行的元素所在的列数一定是确定的，在某一列的元素所在的行数也是一定确定的，我们可以由此来还原原矩阵

# 代码

# JC0106. Reachable Numbers

# 题目大意

给你一个函数 f(x)f(x)f(x)，作用为将 x+1x+1x+1 后，去掉末尾的所有 0，现在给定一个数 nnn，求对 nnn 进行多次 f(n)f(n)f(n) 操作所能得到多少不同的数，包括 nnn 自己

# 思路

不难发现，对于给定的数 nnn，我们最多操作次数不超过 nnn 的位数 * 8+3，所以我们可以不带脑子大胆模拟

# 代码

# JC0107. Collecting Packages

# 题目大意

给出平面直角坐标系上的 nnn 个坐标，要求你从原点出发，在每次移动只能向上或向右移动一个单位长度的情况下是否能经过给出的所有坐标

# 思路

先对给出的坐标按 xxx 轴从小到大排序，yyy 轴也从小到大排序，然后优先向右移动，再向上移动，如果我们发现原先经过的最大 yyy 值大于当前 yyy 值时，则无解

# 代码

# JC0108. Yet Another Crosses Problem

# 题目大意

给定一个矩阵，矩阵有白有黑，求至少将几个白格子涂成黑色才能使某一行某一列全为黑

# 思路

模拟，找出每行每列的最少白色块数，当所涂的格子包含在所需要的行和列之中，答案要减去 1

# 代码

# JC0110. System Testing

# 题目大意

有 nnn 份代码，评测姬可以同时评测 kkk 份代码，第 iii 份代码的测试点数量为 aia_iai​，每个测试点评测需要 1 秒钟，定义 d=⌊100∗mn+0.5⌋d = \lfloor \frac{100*m}{n}+0.5 \rfloord=⌊n100∗m​+0.5⌋，mmm 是已经评测完的代码数量，统计若某代码正在评测第 ppp 个评测点且 p=dp=dp=d 的数量

# 思路

直接模拟，暴力枚举每一秒，复杂度 O(an2k)O(a \frac{n^2}{k})O(akn2​)，最多 1.5∗1081.5*10^81.5∗108，CF 神机能过 CCF 就不一定了

# JC0111. Busy Robot

# 题目大意

你有一个初始位置在数轴原点上的机器人，每秒他能移动一个单位长度

给定 nnn 条指令，每个指令的格式为在第 tit_iti​ 秒移动到 xix_ixi​，如果下达指令时机器人在移动，指令无效

定义当且仅当第 [ti,ti+1][t_i,t_{i+1}][ti​,ti+1​] 秒内机器人到达过 xix_ixi​，求有多少指令是有效的

注意： ti+1t_{i+1}ti+1​ 设为正无穷

# 思路

题中数据让我们放弃了暴力枚举秒数的想法，但是我们可以维护

  • 机器人停止移动的时间
  • 机器人收到指令的时刻
  • 机器人开始移动的位置
  • 机器人停止移动的位置

我们遍历指令，若当前读到的 tit_iti​ 大于等于我们维护的机器人停止移动的时间，说明我们可以执行这条指令并更新以上维护的四种状态，边读入指令边统计答案即可

# 代码

判断指令有效的条件看起来繁琐，实际上只是针对机器人向左 / 向右走进行一个判断，然后给出目标时间段机器人走过的路径的做右端点然后再进行判断就可以了

# JC0113. 时间复杂度

# 题目大意

给出多个以 FFF 或 EEE 开头语句，FxyzF \, x \, y \, zFxyz 表示 for(int x=y;x<=z;x++) ，当 z=nz=nz=n 是，该循环复杂度为 O(n)O(n)O(n)，EEE 表示最近的一个循环结束，循环结构可以进行嵌套使复杂度更高，现给出若干语句和一个时间复杂度，问这些语句的复杂度是否与给出的时间复杂度一样

注意，当给出的语句有以下语法错误时，输出 ERRERRERR

  • F和EF和EF和E 不匹配
  • 新建的变量已经存在

# 思路

模拟，边读入语句边进行模拟，只有 z=n&&y!=nz=n \&\& y!=nz=n&&y!=n 的情况下，该循环复杂度为O(n)O(n)O(n)，其余情况当作 O(1)O(1)O(1) 处理，对于语法错误，我们判断 F和EF和EF和E 的数量是否一样即可，新建变量可以用桶和单调栈记录

# 代码
