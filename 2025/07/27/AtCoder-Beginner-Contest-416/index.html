# D - Match, Mod, Minimize 2

我们先把两个序列的数对 MMM 取模，这样我们只需要求出使原式取模次数最多的方案

对于一个数 xxx ，至少需要一个大于等于 M−xM - xM−x 的数才能使取模次数增加

不妨将任意一个序列排序并从大到小枚举这个序列的数 xix_ixi​ ，找在另一个序列的最小的大于等于 M−xiM - x_iM−xi​ 的数

# E - Development

1≤N≤5001 \leq N \leq 5001≤N≤500

显然考虑 FloydFloydFloyd

先用 FloydFloydFloyd 将两点间距离 dp[u][v]dp[u][v]dp[u][v] 求出来

对于操作 1，直接改变 dp[u][v]dp[u][v]dp[u][v] 和 dp[v][u]dp[v][u]dp[v][u] 的值并跑一遍更新，这是经典做法

对于操作 2，如果单纯每次加点就对每个更新的边重新跑 n2n^2n2 的更新，复杂度是不允许的，但是注意到机场间是互连的，我们可以用一个源点 n+1n+1n+1 表示经过了机场，这样更新点 uuu 的时候只用更新 dp[u][n+1]dp[u][n+1]dp[u][n+1] 和 dp[n+1][u]dp[n+1][u]dp[n+1][u] ，转换成了操作 1

注意这样做会导致坐飞机的时间加倍，因此我们可以把公路时间也加倍，最后统计答案的时候除以 2 就可以了

对于操作 3，直接输出即可

总复杂度 O(QN2+N3)O(QN^2+N^3)O(QN2+N3)​

注意：输入中包含重边，记得取 min
